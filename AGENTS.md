# AGENTS.md

Conventions for this repository. Read this before making changes — human or AI.

## Non-negotiables

1. **No hardcoded user-visible strings.** All copy goes through `i18n/translations.ts` + `t()`. If you add text in a component, you must add an `en` **and** `es` entry. TypeScript will fail the build if a key is missing from either locale.
2. **No emoji in code, commit messages, or UI** unless the user explicitly asked.
3. **Run `pnpm lint` before declaring work done.** It must exit 0.
4. **Respect `prefers-reduced-motion`.** Wrap non-essential animation in `gsap.matchMedia('(prefers-reduced-motion: no-preference)')` or use Framer Motion's `useReducedMotion()`.
5. **Never push to remote without being asked.** Never force-push. Never `git commit --amend` a shared commit.
6. **Never bypass git hooks** (`--no-verify`, `--no-gpg-sign`) unless told to.
7. **No IDE or assistant branding** in branch names, commits, PRs, comments, or docs. Do not use names like “Cursor”, “Copilot”, or “generated by …” in those places. See `.cursor/rules/no-tooling-branding.mdc`.

## Architecture at a glance

- **Two apps, one bundle.** `App.tsx` is the portfolio single-page experience. `BlogApp.tsx` is the React-Router-driven blog. `index.tsx` chooses between them based on the URL.
- **Chapters, not components, drive the homepage.** Each file in `components/chapters/` is a full-bleed section of the scroll narrative. They are rendered in order inside `App.tsx`.
- **Chapters are numbered in their kickers** (`01 — Overview`, `02 — About`, … `08 — Let's connect`). If you insert or remove a chapter, re-number the kickers consistently.
- **GSAP owns scroll. Framer Motion owns discrete UI.** Don't mix.
- **Lenis is the scroll engine.** `window.scrollTo`, CSS `scroll-behavior`, and raw anchor jumps will feel wrong. Use `useSmoothScroll()` → `lenis.scrollTo(...)` for programmatic navigation.

## File layout rules

| Put it here | When it is |
|---|---|
| `components/chapters/` | A full-width scroll-narrative section on the homepage |
| `components/cards/` | A bento-grid card that opens a DetailView modal |
| `components/sections/` | A detail section rendered inside DetailView |
| `components/` (root) | A shared primitive (Grain, SmoothScroll, Globe, SEO, …) |
| `pages/blog/` | Anything blog-specific |
| `content/posts/` | Markdown source for a blog post |
| `scripts/` | Node build-time scripts (no runtime code) |
| `config/seo.ts` | Canonical URLs, schema builders, site metadata |
| `i18n/translations.ts` | All user-facing copy |

## Coding style

### React
- **Function components only.** No class components except `ErrorBoundary`.
- **Prefer `useGSAP({ scope, dependencies })`** over bare `useLayoutEffect` + GSAP — it handles cleanup and React Strict Mode.
- Refs for DOM access (`useRef<HTMLElement>`), not `querySelector` unless you're batching.
- `React.FC<Props>` is fine but not required. Plain `({ ... }: Props)` is also accepted.

### TypeScript
- Strict mode is on. Don't widen types to `any` without a comment justifying it.
- Shared types go in `types/`. Component-local types stay in the component file.
- Use `type` for unions, `interface` for object shapes you might extend.

### Tailwind
- **Mobile-first.** Base styles target phones; add `sm:`, `md:`, `lg:`, `xl:`, `3xl:` as you scale up.
- **Use design tokens**, not raw hex. `text-text-main`, `bg-page`, `border-border`, `text-primary`. See `tailwind.config.js`.
- **Never use `overflow-hidden` on an ancestor of `position: sticky`** — it breaks sticky silently. Use `overflow-clip` instead.
- **Avoid `leading-[0.8]`-style line-heights on display type with descenders.** They clip `g`, `p`, `y` bottoms. Use `leading-[0.95]` or add `pb-[0.08em]`.

### Comments
- Write comments that explain *why*, not *what*. No narrating:
  ```ts
  // ❌  // increment the counter
  counter++;

  // ✅  // Compensate for the Lenis raf tick that lands between frames
  counter++;
  ```
- Don't leave "changed X" comments — git does that.

## Animation playbook

### GSAP + SplitText
- **Don't use `mask: 'words'` / `mask: 'chars'` / `mask: 'lines'`** on display type. The overflow-hidden wrappers slice descenders.
- **Do use plain `type: 'words'`** and animate `y: 40, opacity: 0` → natural. Reserve `yPercent: 110` for masked pill-style reveals where clipping is intentional.
- Register plugins once per file: `gsap.registerPlugin(ScrollTrigger, SplitText);`
- Always revert splits on cleanup: `return () => split.revert();`.

### ScrollTrigger
- **Prefer `once: true`** over `toggleActions` for reveal animations that never need to re-fire.
- Use `ScrollTrigger.batch(...)` only when elements share identical animations *and* users scroll through them linearly. Jump-navigation (TOC clicks) skips batches.
- Call `ScrollTrigger.refresh()` after fonts/images load — handled globally in `SmoothScroll.tsx` via `window.load`.
- Never stack `scrub` on a `matchMedia` that also adds a tween on the same target — double-smoothing causes visible stutter.

### Lenis programmatic scroll
```ts
const { lenis } = useSmoothScroll();
lenis?.scrollTo(target, {
  offset: -88,                           // clear fixed top chrome
  duration: 0.7,
  easing: (t) => 1 - Math.pow(1 - t, 3), // cubic out — crisp, not floaty
});
```
Before a programmatic scroll that crosses reveal-gated content, **kill the pending ScrollTriggers and `gsap.set` the final state** — otherwise dozens of tweens fire as the scroll flies past.

### Framer Motion
- Use `m.*` (not `motion.*`) with `LazyMotion` + `domAnimation` for smaller bundles.
- Every `m.*` that animates in should branch on `useReducedMotion()`.

## i18n patterns

```tsx
import { useLanguage } from '../i18n/LanguageContext';

export function Thing() {
  const { t, language } = useLanguage();

  // ✅  use t()
  return <h2>{t('writingHeadline')}</h2>;

  // ❌  inline ternary
  return <h2>{language === 'es' ? 'Palabras' : 'Words'}</h2>;
}
```

When you need to branch on language for non-copy reasons (date locale, URL suffix, schema), inline ternaries are fine:

```ts
new Date(iso).toLocaleDateString(language === 'es' ? 'es-ES' : 'en-US', ...)
```

## Accessibility

- Every interactive element has a discernible name — `aria-label` or visible text.
- Preserve focus styles; don't `outline: none` without a replacement.
- Heading order is linear (`h1 → h2 → h3`) per page.
- Respect reduced-motion (see above).
- Color contrast: the design tokens are tuned for AA contrast in both themes. Don't introduce raw colors that break this.

## SEO rules

- Every page component uses `<SEO ... />` with `title`, `description`, `canonical`.
- Blog posts must provide `alternates` with `hrefLang: 'en' | 'es' | 'x-default'`.
- JSON-LD schemas are built with helpers in `config/seo.ts` — don't hand-roll them.

## Dependencies

- Runtime deps go in `"dependencies"`. Build-only stuff (vite plugins, TS types) goes in `"devDependencies"`.
- When adding a dependency, use `pnpm add <name>` — never hand-edit `package.json` without running the installer.
- Don't upgrade React, Vite, or GSAP major versions without confirming with the user first.

## Commit style

- Conventional commits (`feat:`, `fix:`, `docs:`, `refactor:`, `chore:`).
- Subject ≤ 50 chars.
- Body only when the *why* isn't obvious from the diff.
- Commit small, scoped units. One concern per commit.

## Pull requests

- Branch naming: `feat/short-summary`, `fix/bug-description`, `docs/what-changed`.
- PR description must include:
  1. What & why (1–3 bullets).
  2. Screenshots/clips for any visual change.
  3. Test plan (checklist of what you verified).

## Where to ask questions

If you're an AI agent and the spec is ambiguous:
1. Check `README.md` for user-facing explanations.
2. Check this file for conventions.
3. Grep the codebase for prior art before inventing a new pattern.
4. Ask the user — don't guess on destructive decisions (schema changes, removals, dep upgrades).
